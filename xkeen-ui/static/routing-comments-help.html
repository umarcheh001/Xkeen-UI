Маршрутизация Xray (05_routing.jsonc)

Эта страница объясняет, как правильно писать комментарии и правила в файле 05_routing.jsonc, чтобы панель XKeen UI могла его загрузить и применить.

Кратко
  • Поддерживаются комментарии: //, #, /* ... */.
  • При сохранении комментарии удаляются, а файл проверяется как обычный JSON.
  • После удаления комментариев не должно быть синтаксических ошибок (висячих запятых, незакрытых скобок и т.п.).

1. Виды поддерживаемых комментариев

1.1. Однострочный комментарий //

// Комментарий целой строки
{
  "rules": [
    // Комментарий перед правилом
    { "type": "field", "outboundTag": "proxy" } // Комментарий в конце строки
  ]
}

1.2. Однострочный комментарий #

# Комментарий целой строки
{
  "domainStrategy": "AsIs",  # Комментарий в конце строки
  "rules": [
    # комментарий внутри списка
    { "type": "field", "outboundTag": "direct" }
  ]
}

1.3. Многострочный комментарий /* ... */

{
  /*
    Большой блок комментария:
    - можно описать, что делает конфиг
    - кто его правил
    - когда вносились изменения
  */
  "rules": [
    {
      "type": "field",
      "ip": ["geoip:private"],
      "outboundTag": "direct"
    }
  ]
}

Важно: каждый многострочный комментарий должен быть закрыт */. Незакрытый блок-сomментарий сломает весь файл.

2. Типичные ошибки

2.1. Висячая запятая перед закрывающей скобкой

После удаления комментариев JSON не должен содержать лишних запятых.

Неправильно:

{
  "rules": [
    { "type": "field", "outboundTag": "proxy" }, // последний элемент,
  ] # после удаления комментариев останется лишняя запятая
}

Правильно:

{
  "rules": [
    { "type": "field", "outboundTag": "proxy" } // последний элемент, без запятой
  ]
}

2.2. Незакрытый многострочный комментарий

Неправильно:

{
  /* Начали писать
     большой комментарий и забыли закрыть
  "rules": [
    ...
  ]
}

Правильно:

{
  /* Всё ОК */
  "rules": [
    ...
  ]
}

2.3. Комментарии внутри строк

Последовательности //, # и /* внутри строк не считаются комментариями:

{
  "note": "это просто текст // а не комментарий"
}

3. Как устроены правила маршрутизации

Внутри поля "rules" каждый объект — это отдельное правило. Основные поля:

  • "type": "field" — тип правила (обычно его не меняют).
  • "domain": [...] — список доменов или групп доменов (geosite:...).
  • "ip": [...] — список IP или групп IP (geoip:...).
  • "outboundTag": "proxy" — тег, который должен существовать в 04_outbounds.json.

Пример: отправить Netflix через прокси:

{
  "type": "field",
  "domain": [
    "geosite:netflix",
    "netflix.com"
  ],
  "outboundTag": "proxy"
}

Пример: локальные сети в обход прокси:

{
  "type": "field",
  "ip": [
    "geoip:private"
  ],
  "outboundTag": "direct"
}

4. Готовый пример 05_routing.jsonc

Ниже — рабочий шаблон 05_routing.jsonc, который можно использовать как базу и редактировать под себя.

/*
 * 05_routing.jsonc — пример с комментариями
 *
 * Поддерживаются комментарии:
 *   // однострочный комментарий
 *   # однострочный комментарий
 *   /* многострочный комментарий *\/
 *
 * После удаления комментариев файл превращается в обычный валидный JSON.
 * Следи только за тем, чтобы не было "висячих" запятых после последнего элемента.
 */

# Общие настройки маршрутизатора Xray
{
  // Как обрабатывать домены (AsIs / IPIfNonMatch / IPOnDemand)
  "domainStrategy": "AsIs",

  // Как сопоставлять домены (plain / vdomain / mph / hybrid)
  "domainMatcher": "hybrid",

  // Основные правила маршрутизации
  "rules": [
    // ===================== ЛОКАЛЬНЫЕ СЕТИ / ЛОКАЛЬНЫЕ ДОМЕНЫ =====================

    {
      // Весь приватный диапазон IP → direct
      "type": "field",
      "ip": [
        "geoip:private",         // RFC1918, локальные сети
        "geoip:cn"               // пример: Китай напрямую (можно удалить, если не нужно)
      ],
      "outboundTag": "direct"    // тег должен существовать в 04_outbounds.json
    },

    {
      // Локальные домены, типа router.local, *.lan и т.п.
      "type": "field",
      "domain": [
        "geosite:private"
      ],
      "outboundTag": "direct"
    },

    // ===================== БЛОКИРОВКА РЕКЛАМЫ / НЕНУЖНЫЙ ТРАФИК =====================

    {
      // Рекламные домены по готовым спискам
      "type": "field",
      "domain": [
        "geosite:category-ads-all"  // списки рекламы (если импортированы в geosite)
      ],
      "outboundTag": "block"        // тег block должен быть настроен в outbounds
    },

    // ===================== СТРИМИНГ / МЕДИА ЧЕРЕЗ ПРОКСИ =====================

    {
      /*
       * Стриминговые сервисы → proxy
       * сюда можно добавлять свои геосайты и домены
       */
      "type": "field",
      "domain": [
        // Сервисы из geosite
        "geosite:netflix",
        "geosite:disney",
        "geosite:youtube",

        // Прямые домены (пример)
        "netflix.com",
        "youtube.com"
      ],
      "outboundTag": "proxy"
    },

    // ===================== РАБОТА / VPN ДЛЯ ОФИСА =====================

    {
      /*
       * Пример: рабочие ресурсы через отдельный прокси
       * Можно удалить или раскомментировать и дописать свои домены.
       */
      "type": "field",
      "domain": [
        "corp.example.com",
        "jira.example.com",
        "vpn.example.com"
      ],
      "outboundTag": "work-proxy"   // не забудь добавить этот тег в outbounds
    },

    // ===================== ПРИМЕРЫ ДЛЯ ПОЛЬЗОВАТЕЛЕЙ =====================

    /*
    {
      // Пример: игры в Steam идут напрямую
      "type": "field",
      "domain": [
        "store.steampowered.com",
        "api.steampowered.com"
      ],
      "outboundTag": "direct"
    },
    */

    /*
    {
      // Пример: какие-то сайты всегда через VPN
      "type": "field",
      "domain": [
        "mybank.com",
        "secure.example.org"
      ],
      "outboundTag": "proxy"
    },
    */

    // ===================== ПРАВИЛО ПО УМОЛЧАНИЮ =====================

    {
      /*
       * Фоллбэк: всё, что не попало ни под одно правило выше,
       * идёт через proxy (можно сменить на direct, если нужно).
       */
      "type": "field",
      "outboundTag": "proxy"
    }
  ]
}

5. Советы

  • Если сомневаетесь — закомментируйте правило, а не удаляйте его.
  • После правок убедитесь, что:
    • все скобки {} и [] закрыты;
    • строки в двойных кавычках "...";
    • нет лишней запятой перед ] или }.

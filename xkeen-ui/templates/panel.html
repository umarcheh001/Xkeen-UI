Содержание

            Как это устроено
            🚀 Быстрый старт
            Серверы вручную
            Подписки Xray (детали)
            Балансировщик
            Правила
            Исключения (🔒)
            Ошибки и подсказки
          


          

            

              Идея простая: добавьте серверы (outbounds) с разными tag,
              создайте балансировщик, в правилах укажите balancerTag.
              Дальше Xray сам выбирает “какой outbound использовать сейчас”.
            


            
Как это устроено в панели

            
    
              
  • 04_outbounds.json — ручные серверы (каждый с уникальным tag).
    • 
              
  • 04_outbounds.<id>.json — generated-фрагменты подписок. Основной 04_outbounds.json не перезаписывается.
    • 
              
  • 05_routing.json — routing.balancers и routing.rules.
                (в некоторых сборках/профилях может быть 05_routing_hys2.json; панель показывает активный файл в селекторе сверху)
              
    • 
              
  • 07_observatory.json — измерение доступности/пинга (нужно для leastPing).
    • 
            

            

              Минимальный пример (для понимания)
              
Смысл: несколько outbounds с тегами → один balancer → правило с balancerTag.

              

                // routing.balancers

                { "tag": "proxy", "selector": ["vless-"], "strategy": {"type":"leastPing"}, "fallbackTag": "direct" }


                // routing.rules (в конце списка)

                { "type": "field", "balancerTag": "proxy" }
              

            


            

              Keenetic: ядро Xray ставится пакетом xkeen через Entware. Перезапуск — xkeen -restart, панель делает сама. Xray применяет конфиг только после рестарта. В мастерах “Применить + Рестарт” это происходит автоматически (job + лог).
            


            
🚀 Быстрый старт: подписка + балансировщик за 5 минут

            

              Самый простой путь для Keenetic-пользователя
              
1. Создайте подписку. Карточка Прокси → кнопка Подписки. Заполните:

              
    
                
  • Имя — любое (например, VPN-1).
    • 
                
  • Tag prefix — короткий префикс, например sub. Generated-теги станут sub--node1, sub--node2 и т.д.
    • 
                
  • URL подписки — HTTP(S) ссылка от провайдера (поддерживаются share-ссылки, base64, Xray JSON).
    • 
                
  • (Опц.) Фильтр имени — regex, оставить только нужные локации: Germany|Netherlands|SG.
    • 
                
  • (Опц.) Фильтр типа — vless|trojan|vmess|ss|hy2.
    • 
                
  • (Опц.) Фильтр транспорта — для VLESS: xhttp|ws|grpc|httpupgrade|tcp.
    • 
                
  • Интервал — 1-168 ч (6 ч — разумно).
    • 
                
  • Галки: Авто (плановое обновление), Пинг (добавить теги в observatory — нужно для leastPing), Обновить сразу.
    • 
              

              
Нажмите Сохранить → панель скачает подписку и покажет список узлов. Там можно пингануть каждый узел кнопкой и вручную отключить медленные.


              
2. Создайте балансировщик. Вернитесь в Роутинг Xray → в карточке правил нажмите ⚡:

              
    
                
  • Selector: sub-- (ваш префикс).
    • 
                
  • Стратегия: leastPing.
    • 
                
  • Fallback: direct.
    • 
                
  • Оставьте галку “создать дефолтное правило” — весь трафик пойдёт через балансировщик.
    • 
                
  • Нажмите Применить + Рестарт.
    • 
              


              
3. Готово. Xray перезапустился, трафик идёт через подписку по leastPing.


              

                💡 Чтобы пускать через VPN только часть трафика — снимите галку дефолтного правила и в разделе “Правила” добавьте rules только для нужных доменов.
              

              

                ⚠️ Keenetic-лимит: 300+ активных узлов в балансировщике кладут роутер (RAM / observatory / conntrack). Держите десятки, а не сотни — используйте regex-фильтры и ручные исключения узлов.
              

            


            
Серверы в 04_outbounds вручную (опционально)

            

              Если не используете подписки — добавьте серверы руками
              

                Откройте карточку Outbounds (04_outbounds.json) и нажмите 🧱 Пул прокси.
                Вставьте несколько строк со ссылками — по одной записи на строку:
              

              

                vless-1 | vless://....

                vless-2 = vless://....

                vless://....#vless-3
              

              
    
                
  • Рекомендуется общий префикс в тегах (vless-*) — проще настраивать selector.
    • 
                
  • Теги direct/block/dns — служебные, их не занимайте.
    • 
                
  • После сохранения пул попадёт в 04_outbounds. В мастерах теги подтянутся автоматически.
    • 
              

            


            
Подписки Xray (подробно)

            

              Как работают подписки внутри
              
Подписка = URL, по которому провайдер отдаёт список VLESS/VMess/Trojan-ссылок. Панель:

              
    
                
  1. Скачивает по расписанию (интервал 1-168 ч).
    2. 
                
  2. Парсит ноды (share-ссылки / base64 / Xray JSON).
    3. 
                
  3. Пишет отдельный файл 04_outbounds.<id>.json с тегами <prefix>--<node>.
    4. 
                
  4. (Опц.) Добавляет теги в 07_observatory.json для leastPing.
    5. 
              

              

                Главное: подписки не трогают ручной 04_outbounds.json. Они живут отдельными фрагментами; мастер leastPing видит теги из всех outbounds-файлов.
              

            

            

              Фильтры — уменьшают число узлов перед сохранением
              
    
                
  • Имя (regex): Germany|Netherlands|SG — оставить только нужные локации.
    • 
                
  • Тип (regex): vless|trojan|vmess|ss|hy2 — убрать лишние протоколы.
    • 
                
  • Транспорт (для VLESS): xhttp|ws|grpc|httpupgrade|tcp — оставить только нужные транспорты.
    • 
              

              

                Regex чувствителен к регистру в зависимости от исходной подписки. Если сомневаетесь — используйте короткие явные токены через |.
              

            

            

              Список узлов: ping и ручные исключения
              
    
                
  • Кнопка ping у каждого узла показывает реальную задержку — удобно находить медленные.
    • 
                
  • Галочки — вручную отключить конкретные узлы: они не попадут в generated-файл при следующем обновлении.
    • 
              

              

                Управление списком подписок — в модалке Прокси → Подписки: создать / редактировать / обновить / удалить.
              

            


            
Балансировщик

            

              ⚡ Быстрый старт — рекомендовано для большинства
              

                Кнопка ⚡ создаёт/обновляет балансировщик со стратегией leastPing и настраивает всё нужное:
              

              
    
                
  • Создаёт routing.balancers с strategy.type = leastPing.
    • 
                
  • Заполняет selector по списку outbound‑тегов (можно отредактировать).
    • 
                
  • Генерирует 07_observatory.json, чтобы Xray мог измерять задержку.
    • 
                
  • Опционально добавляет “дефолтное” правило match‑all с balancerTag (чтобы весь трафик пошёл через балансировщик).
    • 
                
  • Нажатие “Применить + Рестарт” — сохранит файлы и перезапустит Xray в job‑режиме с логом.
    • 
              

              

                Если не хотите, чтобы весь трафик шёл через балансировщик — снимите галочку про “дефолтное правило” и создайте правило вручную только для нужных доменов/устройств.
              

            

            

              ⚖️ Добавить балансировщик — ручная настройка
              

                Кнопка ⚖️ добавляет пустой балансировщик в список “Балансировщики”. Дальше вы редактируете поля:
              

              
    
                
  • tag — имя балансировщика (его потом указывают в balancerTag у правил).
    • 
                
  • selector — список префиксов outbound‑тегов. Xray выберет все outbounds, чьи tag начинаются с этих строк.
    • 
                
  • strategy — алгоритм: leastPing / leastLoad / random.
    • 
                
  • fallbackTag — запасной outbound (обычно direct).
    • 
              

              

                selector — префикс, не точный тег! Если у вас теги vless-1 и vless-10, selector vless-1 захватит оба. Делайте теги уникальными без пересечений.
              

            


            
Правила маршрутизации

            

              Как отправить трафик в балансировщик
              

                В правилах (routing.rules) у каждого правила должно быть одно из двух:
                outboundTag (на конкретный сервер) или balancerTag (в балансировщик).
              

              
    
                
  • Чтобы использовать балансировщик — задайте balancerTag: <tag балансировщика>.
    • 
                
  • Правила проверяются сверху вниз. Первое совпавшее побеждает.
    • 
                
  • Дефолтное правило (match‑all) ставят в самый низ списка.
    • 
              

              

                { "type": "field", "balancerTag": "proxy" }
              

              

                Если правила/балансировщики редактируете через карточки: 💾 Применить изменения → в редакторе Сохранить → (при необходимости) Перезапуск. В мастере ⚡ это делается одной кнопкой.
              

            


            
Исключения (🔒 Принудительные правила)

            

              Когда нужны принудительные правила
              

                Если какие‑то домены/IP нельзя отправлять в балансировщик (локальные ресурсы, банки, IPTV, админка роутера) — используйте кнопку 🔒.
              

              
    
                
  • Мастер создаёт правила type: field, ведущие на конкретный outboundTag (обычно direct).
    • 
                
  • Правила вставляются выше правила с balancerTag, поэтому они “обходят” балансировщик.
    • 
                
  • Опция “Только для redirect/tproxy” — не трогать другие inbound’ы.
    • 
              

              

                geosite:category-ru

                domain:example.com

                geoip:private
              

            


            
Ошибки и подсказки

            

              Пустой список тегов в мастере
              
    
                
  • В 04_outbounds должны быть outbounds с уникальными tag.
    • 
                
  • Нажмите “⟳ Обновить список” в мастере (фильтрует служебные теги).
    • 
                
  • Если теги должны быть из подписки — проверьте, что подписка реально скачалась (кнопка “Обновить” в модалке подписок, посмотрите ошибки).
    • 
              

            

            

              leastPing не работает / не видно переключений
              
    
                
  • Для leastPing нужен 07_observatory.json и перезапуск Xray.
    • 
                
  • Используйте ⚡ мастер и кнопку “Применить + Рестарт” — создаёт observatory и показывает лог.
    • 
                
  • Проверьте, что серверы реально доступны из роутера (DNS / маршруты / время/NTP).
    • 
              

            

            

              Трафик “не туда” / правило не срабатывает
              
    
                
  • Правила сверху вниз. Исключения должны быть выше дефолта.
    • 
                
  • Проверьте условия rule: domain / ip / source / inboundTag.
    • 
                
  • selector — префиксы, не точные теги. Не делайте пересекающиеся.
    • 
              

            

            

              Роутер тормозит / Xray падает после подписки
              
    
                
  • Keenetic ограничен по RAM (обычно 128-256 МБ) и CPU. Большие подписки нагружают observatory и conntrack.
    • 
                
  • Сужайте подписку фильтрами (имя / тип / транспорт) или вручную отключайте лишние узлы.
    • 
                
  • 300+ активных узлов в балансировщике — подтверждённый случай падения роутера.
    • 
              

            

            

              Подписка не обновляется сама
              
    
                
  • Включите чекбокс Авто в форме подписки.
    • 
                
  • Интервал — 1-168 часов.
    • 
                
  • Автообновление идёт через панель — она должна быть запущена.
    •

      Справка: файловый менеджер
      
    

    

      

        
Двухпанельный менеджер (как Total Commander): действия применяются к активной панели и текущему выделению.


        
Панели

        
    
          
  • Target: local — файлы на роутере (в пределах разрешённых корней), remote — подключение к серверу (SFTP/FTP/FTPS).
    • 
          
  • Connect…/Disconnect — открыть/закрыть remote-сессию.
    • 
          
  • Поле пути: введите путь и нажмите Enter для перехода.
    • 
          
  • ↑ — перейти в родительский каталог, ⟳ — обновить список.
    • 
        


        
Выделение и навигация

        
    
          
  • Клик — фокус + выделение; двойной клик или Enter — открыть файл/папку.
    • 
          
  • ↑/↓ — перемещение фокуса по списку.
    • 
          
  • Ctrl (или клик по чекбоксу) — добавить/убрать из выделения; Shift — выделить диапазон.
    • 
          
  • ПКМ — контекстное меню (открыть/скачать/копировать/переместить/переименовать/удалить/создать…).
    • 
        


        
Операции

        
    
          
  • ➕ Папка / ➕ Файл — создание в активной панели.
    • 
          
  • ⬆ Upload — загрузка файлов с компьютера в активный каталог.
    • 
          
  • ⬇ Download — скачать выделенное. Для папки всегда будет .zip. Для нескольких файлов можно выбрать: .zip или по очереди (каждый файл отдельно).
    • 
          
  • F5 — копировать из активной панели в другую, F6 — переместить.
    • 
          
  • Если в назначении есть файлы с теми же именами — появится окно «Конфликты» (заменить/пропустить).
    • 
          
  • F2 — переименовать (только в текущем каталоге).
    • 
          
  • F8 — в корзину (в корзине — удалить навсегда).
    • 
          
  • Drag&Drop: перетаскивание между панелями (по умолчанию move; удерживайте Ctrl для copy) или выбор действия в диалоге.
    • 
          
  • Операции — список фоновых задач (копирование/перемещение/удаление/загрузка) и их прогресс.
    •

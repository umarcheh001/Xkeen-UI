              Справка: Custom CSS
              Как пользоваться редактором, как откатиться (Safe mode) и примеры готовых правок.
            
    
            
          
    
          
    
            
    
              
    
	                Этот редактор сохраняет файл UI_STATE_DIR/custom.css на роутере и подключает его на всех страницах (включая /login и /setup).
	                На роутере UI_STATE_DIR обычно /opt/etc/xkeen-ui (можно переопределить через env XKEEN_UI_STATE_DIR / XKEEN_UI_DIR). При желании можно править файл и напрямую по SSH: UI_STATE_DIR/custom.css.
	                Файл отдаётся по URL /ui/custom.css и подключается с версией ?v= (обновляется при сохранении).
	                
    
	                Если что-то “сломалось” — откройте любую страницу с ?safe=1 (также подходят safe=true/yes/on): тогда custom.css не загрузится.
	                Safe mode выключает только global Custom CSS; custom_theme.css и локальные настройки интерфейса (типографика/лейаут) остаются.
	                
    
	                Ограничения: до 250 000 символов. Отключение хранится файлом-флажком UI_STATE_DIR/custom_css.disabled (кнопка Disable).
              
    

              
    
                
    
                  
                  
                
    
                
    
              
    


              
    
                Быстрый старт
                
    
                  
      
                    
    1. Впишите CSS в редактор.
      2. 
                    
    2. Нажмите Save — сохранит и включит.
      3. 
	                    
    3. Если эффекта нет — обновите страницу (Ctrl+F5) или проверьте, что в адресе нет параметра safe.
      4. 
                    
    4. Disable — выключает применение (файл остаётся).
      5. 
                    
    5. Reset — удаляет CSS и сбрасывает всё.
      6. 
                  
    
                
    
              
    

              
    
                Как найти нужный селектор
                
    
                  
      
                    
    1. Откройте панель в браузере → нажмите правой кнопкой по элементу → Inspect / Исследовать.
      2. 
                    
    2. Вкладка Elements покажет HTML и классы (например .card, .btn-secondary).
      3. 
                    
    3. Вкладка Styles показывает, какие правила применяются и чем их можно переопределить.
      4. 
                    
    4. Лучше “сужать” селектор: вместо div используйте классы/ID, чтобы не ломать всё подряд.
      5. 
                  
    
                
    
              
    

              
    
                Слои, темы и приоритет
                
    
                  
      
                    
    • База: стили из /static/styles.css + стили конкретной страницы.
      • 
	                    
    • Локальные настройки интерфейса: блок DevTools → “Интерфейс” сохраняется в браузере и применяет правки через data-* на <html> (например data-xk-compact, data-xk-hide-hints, data-xk-container) и через inline CSS‑переменные (--xk-font-scale, --xk-container-max-width и т. д.). Если он включён, он может перекрыть ваши переменные: либо сбросьте его кнопкой Reset, либо используйте !important для переменных.
      • 
                    
    • Theme editor: если вы сохраняли тему в DevTools → “Тема и CSS‑переменные”, подключается /ui/custom-theme.css (файл UI_STATE_DIR/custom_theme.css).
      • 
                    
    • Custom CSS: этот редактор управляет /ui/custom.css (файл UI_STATE_DIR/custom.css). Он подключается последним и обычно перекрывает всё остальное.
      • 
                    
    • Если правило не “берётся” — чаще всего нужно либо сделать селектор более специфичным, либо добавить !important.
      • 
                    
    • Текущая тема хранится в атрибуте data-theme на <html>: dark или light. Можно писать правила выборочно:
                      html[data-theme="dark"] ... и html[data-theme="light"] ....
      • 
                  
    
                
    
              
    

              
    
                Примеры (копируйте и правьте)
                
    
                  Ниже — набор типовых твиков. Каждый пример можно вставить целиком или по частям.
                
    

	                
    1) Увеличить шрифт глобально (через CSS‑переменные)
    
	                
    
	                  Если вы меняли DevTools → “Интерфейс → Типографика”, там значения ставятся inline на <html> и могут перекрывать :root из CSS. Для этого примера поставьте там 100% / “как UI” / “по умолчанию” или нажмите Reset.

                Safe mode и аварийный откат
                
    
                  
      
	                    
    • Добавьте ?safe=1 (или safe=true/yes/on) к адресу — страница загрузится без custom.css. Работает везде, включая /login и /setup (если редирект мешает — откройте напрямую /login?safe=1).
      • 
	                    
    • Safe mode отключает только global Custom CSS. Если вы хотите «обнулить» тему — используйте DevTools → “Тема и CSS‑переменные” → Reset global.
      • 
	                    
    • Зайдите в DevTools → Custom CSS → нажмите Disable (временно) или Reset (полный сброс).
      • 
	                    
    • Если “перекрыли” клики (например сделали pointer-events:none) — safe mode поможет зайти обратно.
